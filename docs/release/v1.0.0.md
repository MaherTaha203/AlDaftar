# Release Notes — v1.0.0

**Release date:** 2026-07-06
**Tag:** `v1.0.0` · **Commit:** merge of PR #1 (`production-review`)
**Previous release:** `v1.0.0-rc.1`

## Highlights

First stable release. Promotes Release Candidate `v1.0.0-rc.1` after a full
production review of the running application — every route, form, dialog,
report, print view, and document workflow was driven end-to-end — plus code
audits for financial correctness, data integrity, and security. All confirmed
issues were fixed in PR #1 (owner-reviewed and merged); no defect remains open
within the approved scope.

## What's included

- **Master data:** Suppliers, Products, Categories, Units, Currencies —
  create / edit / archive with active-first Arabic-collated lists.
- **Documents (Draft → Posted, immutable after posting):** Purchases,
  Purchase Returns (returnable-remainder guarded), Payments (running-balance
  model with separate settlement discount).
- **Reports:** 16-report catalog (statements, balances, purchases, payments,
  returns, product movement, attachments, audit) with screen view, print /
  PDF, and Excel (CSV) export.
- **Attachments:** image + PDF archive (10 MB) attached to suppliers and
  documents, with a global library screen.
- **Audit Log:** immutable append-only trail across master data, documents,
  attachments, and settings.
- **Print:** purchase invoice, payment voucher, and return document with
  company header, DD/MM/YYYY dates, and Arabic amount-in-words.
- **Dashboard:** payable total, suppliers with balance, month purchases /
  payments, pending drafts, recent documents.

## Fixed in this release (since v1.0.0-rc.1 — PR #1)

### Correctness & data safety

- **Supplier statement:** a from-date filter no longer drops prior history —
  the opening balance now carries every posted document before the range, so
  the statement always agrees with the balances report
  (07_supplier_statement_effect.md §3.2/§3.4). The closing balance appears in
  the report header, which stays visible even when the range has no rows.
- **Single-tab guard:** the app now enforces its single-writer model with a
  Web Locks–based guard — opening a second tab can no longer silently revert
  posted documents or issue duplicate official numbers; the older tab shows a
  blocking «الدفتر مفتوح في نافذة أخرى» overlay with a continue action.
- **Audit-gap visibility:** if an audit append fails (e.g. storage quota), the
  owner now sees a persistent error toast instead of a console-only warning —
  the trail stays best-effort, but a gap is never silent (BDD-010).

### UI / UX

- **Amount inputs:** the currency/unit label no longer overlaps the first
  digits of the value (500.00 previously rendered as «₪00.00»).
- **Dates:** document lists, document details, and attachment rows now show
  DD/MM/YYYY (BDR-18) like every other surface, instead of raw ISO dates.
- **Posting validation:** posting a purchase without supplier-invoice info
  names the violated field inline («أدخل مرجع فاتورة المورد أو حدد "بدون
  فاتورة مورد"») instead of a generic banner.

### Accessibility & polish

- App icon added (`app/icon.svg`) — pages no longer log a favicon 404.
- The 404 screen has a proper `h1` heading.
- Search boxes and the settings logo file input expose accessible names.

## Known limitations (unchanged, accepted scope)

- **Data durability (TD-004):** records live in one browser's localStorage
  and attachment binaries in IndexedDB — single device, no server backup.
  This is the defining constraint of this release; the Supabase migration is
  the planned next project (`docs/release/NEXT_PROJECT.md`).
- **No authentication:** private single-user model (PD-18); device access =
  full access.
- **Excel export** is a BOM-prefixed CSV, not a styled `.xlsx` (TD-005).
- **Supplier Aging report** deferred pending BDD-009 (buckets + method).

## Upgrade notes

No migration needed from `v1.0.0-rc.1` — the storage schema is unchanged.
Existing localStorage/IndexedDB data is picked up as-is.
